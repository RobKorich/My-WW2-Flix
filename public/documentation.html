Documentation

An App for WW2 movie enthusiasts where they can recieve information on WW2 movies, directors, and specific battles so that they can learn more about the WW2 movies they have seen or wish to see in the future. They will be able to create a profile in order to save data about their favorite movies.

Business Logic URL HTTP Method Request Body Data Format Response Body Data Format
Get a list of all movies to user /movies GET None A JSON object holding data about all the movies

{

"Description": "1970 epic war film that dramatizes the Japanese attack on Pearl Harbor in 1941",
"ImagePath": "ToraToraTora.png"

"Description": "Action film is a genre in which the protagonist or protagonists are thrust into a series of events that typically include violence, extended fighting, physical feats and frantic chases."
},

"Bio": "American film director whose career spanned over four decades, beginning at the height of the Golden Age of Hollywood and lasting through the American New Wave",

}
}
Get data about movie genre to user /movies/genres/[Name] GET None A JSON object holding data about movie genres. Example:

{
"Name": "Action",
"Description": "Action film is a genre in which the protagonist or protagonists are thrust into a series of events that typically include violence, extended fighting, physical feats and frantic chases."
}
Get data about movie director including name, bio, birth year, and death year. /movies/directors/[Name] GET None A JSON object holding data about movie director Example:

{
"Name": "Richard Fleischer",
"Bio": "American film director whose career spanned over four decades, beginning at the height of the Golden Age of Hollywood and lasting through the American New Wave",
"Birth": "1916",
"Death": "2006"
}
Add data to register new user accounts /users POST A JSON object holding data about the user to add Example:

{
"Username" : "RobKorich1234",
"Password" : "password1234",
"Email" : "RobKorich1234@gmail.com",
"Birthday" : "12-09-1985"
}

 A JSON object holding data about the user that was added including an ID. Example:

{
"_id" : 1234567890,
"Username" : "RobKorich1234",
"Password" : "password1234",
"Email" : "RobKorich1234@gmail.com",
"Birthday" : "12-09-1985"
}
Allow user to update their user info (username, password email, and DoB). /users/[Username] PUT A JSON object holding data of updated user information Example:

{
"Username" : "RobKorich1234",
"Password" : "password1234",
"Email" : "RobKorich1234@gmail.com",
"Birthday" : "12-09-1985"
}

 A text message showing user's information has been updated successfully.
Allow user to add movies to their favorite's list /users/[Username]/favorites/[movieID] POST A JSON object holding data about the movie to add to the user's favorite's list A text message showing movie has been added to user's favorites
Allow users to remove a movie from their favorite's list /users/[Username]/favorites/[movieID] DELETE None A text message showing movie was removed from user's favorite's list
Allow users to deregister /users/[Username] DELETE None A text message showing user's account has been deleted
